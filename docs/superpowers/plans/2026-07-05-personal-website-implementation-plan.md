# 中文个人网站实施计划

## 目标

基于已确认的设计方案，搭建一个中文个人网站第一版。第一版使用 Astro，首页是一屏完整展示的手绘收藏柜工作室，提供五个清晰入口：关于我、技术文章、项目入口、个人生活、联系我。

## 阶段 1：项目骨架

创建 Astro 静态网站基础结构：

- `package.json`
- `astro.config.mjs`
- `tsconfig.json`
- `src/layouts/BaseLayout.astro`
- `src/styles/global.css`
- `src/pages/index.astro`
- `src/pages/about.astro`
- `src/pages/posts/index.astro`
- `src/pages/projects/index.astro`
- `src/pages/life/index.astro`
- `src/pages/contact.astro`

验收标准：

- `npm install` 成功。
- `npm run dev` 能启动本地开发服务器。
- `npm run build` 成功。
- 五个页面路径都能访问。

## 阶段 2：首页收藏柜

先用 CSS 绘制一个可用的手绘风收藏柜占位版本，不等待最终 AI 图片。这样可以先验证布局、点击区域和响应式行为。

首页包含：

- 一屏完整展示的收藏柜区域。
- 五个物件入口。
- 中文标签和短说明。
- hover 高亮与轻微浮动。
- 移动端固定显示入口标签。

验收标准：

- 桌面端首屏能看到全部五个入口。
- 移动端入口不重叠，文字可读。
- 所有入口能点击到正确页面。

## 阶段 3：内容集合

建立 Markdown 内容流：

- 技术文章集合：`src/content/posts/`
- 项目集合：`src/content/projects/`
- 生活记录集合：`src/content/life/`
- 内容 schema 包含标题、日期、摘要、标签和封面图字段。

先放入示例内容，便于验证列表页。

验收标准：

- 新增 Markdown 文件后能出现在对应列表页。
- 文章、项目和生活记录列表显示标题、日期、摘要和标签。
- 构建时 schema 校验通过。

## 阶段 4：基础页面

补齐五个一级页面的第一版内容结构：

- 关于我：自我介绍、当前关注方向、技能关键词。
- 技术文章：文章列表。
- 项目入口：项目卡片列表。
- 个人生活：生活记录列表。
- 联系我：邮箱和外部链接占位。

验收标准：

- 页面内容结构完整。
- 空状态或示例内容不显得破碎。
- 导航和返回首页路径明确。

## 阶段 5：AI 手绘资产接入

在网站骨架稳定后，再生成并接入手绘资产。

资产流程：

1. 生成 3 到 5 张收藏柜风格探索图。
2. 选定一张作为主视觉方向。
3. 生成首页收藏柜主图或五个物件局部图。
4. 压缩为 WebP。
5. 将 CSS 占位图替换为真实图片，并保留网页文本标签。

验收标准：

- 图片不包含模型生成的中文文字。
- 图片加载后不破坏热点定位。
- 首屏图片经过压缩，加载体积可接受。

## 阶段 6：验证和交付

验证内容：

- `npm run build`
- 桌面端截图检查。
- 移动端截图检查。
- 入口点击检查。
- 页面文字溢出检查。

交付标准：

- 本地开发服务器可访问。
- 构建成功。
- 首页、五个一级页面、三类内容列表都可用。
- README 包含开发、构建和部署说明。

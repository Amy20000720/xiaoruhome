# 梦幻糖果小镇互动地图实施计划

## 目标

把当前 CSS 收藏柜首页改为参考 `cecibaldoni.github.io` 结构的高质量插画地图首页。首页主视觉是一张 AI 生成的梦幻糖果小镇地图，前端只负责热点、真实中文标签、响应式和内容路由。

## 阶段 1：生成概念插画

使用高质量图像模型生成 3 到 5 张概念图。概念图只用于筛选方向，不直接作为最终交付。

要求：

- 横向网页首页构图。
- 完整的小镇地图，而不是单个风景或散乱图标。
- 五个清晰地点：糖果小屋、奶油图书馆、魔法工坊、甜品咖啡馆、星星邮局。
- 无文字、无字母、无 logo、无水印。
- 风格梦幻、精致、甜美，避免廉价贴纸和粗糙儿童画。

输出：

- 候选图保存到 `public/assets/candy-town/concepts/`。
- 每张图用清晰文件名标识方向。

验收：

- 至少有 1 张候选图达到可继续精修的质量。
- 候选图里五个地点位置足够明确，可以叠加热点。

## 阶段 2：确定主图并处理网页资产

从候选图中选择一张作为主图方向。如果模型结果足够好，直接作为第一版主图；如果仍需精修，先记录需要修正的点。

处理内容：

- 保存源图到 `public/assets/candy-town/source/`。
- 生成网页用 WebP：`public/assets/candy-town/candy-town-map.webp`。
- 控制图片体积，避免首屏过重。

验收：

- 主图没有文字或水印。
- 主图在横向首屏中构图完整。
- 图片可被 Astro 静态资源路径正常访问。

## 阶段 3：首页结构改造

改造 `src/pages/index.astro`：

- 移除当前 CSS 收藏柜结构。
- 使用 `<picture>` 或 `<img>` 渲染糖果小镇主图。
- 在地图上叠加五个 `<a>` 热点。
- 每个热点使用真实中文标签和短说明。
- 添加地图下方备用文字入口，主要服务移动端和可访问性。

热点映射：

- 糖果小屋：`/about`
- 奶油图书馆：`/posts`
- 魔法工坊：`/projects`
- 甜品咖啡馆：`/life`
- 星星邮局：`/contact`

验收：

- 五个热点能点击到正确路由。
- 标签不依赖图片内文字。
- 移动端入口可读、可点。

## 阶段 4：全局视觉系统改造

更新 `src/styles/global.css`：

- 替换当前木色收藏柜色系。
- 建立糖果小镇色彩变量：奶油白、柔粉、薄荷蓝、香草黄、淡紫、少量香槟金、深莓色文字。
- 内容页卡片改为轻盈、精致、梦幻风格。
- 保持技术文章正文可读，不让背景装饰干扰阅读。

验收：

- 首页和内容页视觉统一。
- 文章列表、项目列表、生活列表仍清晰可读。
- 页面没有文字溢出或横向滚动。

## 阶段 5：响应式和热点定位验证

使用本地浏览器验证：

- 桌面端 1280x720。
- 移动端 390x844。
- 至少检查首页、五个一级页面、一个文章详情页。

验收：

- 首页地图在桌面端首屏中完整展示。
- 移动端有完整可用的导航体验。
- 五个热点不重叠，不误点。
- 无横向溢出。

## 阶段 6：构建、提交、部署

执行：

```bash
npm run build
git status --short --branch
git add <changed-files>
git commit -m "Redesign homepage as candy town map"
git push
npm run deploy
```

验收：

- `npm run build` 为 0 errors、0 warnings、0 hints。
- `main` 推送成功。
- `gh-pages` 部署成功。
- 公开地址可以访问更新后的页面。

## 不做事项

- 不继续用 CSS 绘制最终小镇插画。
- 不复制参考站或商业 IP 的素材。
- 不在模型图片中加入中文文字。
- 不为第一版做复杂动画、3D 或滚动叙事。

# Cecilia 风格个人博客改版方案

## 背景

当前网站已经完成 Astro 静态站、Markdown 内容流、GitHub Pages 发布和一张互动地图首页。但现有版本仍以“糖果小镇”为站点口径，视觉和文字设计更像主题插画站，不够接近用户指定参考站 `https://cecibaldoni.github.io/` 的个人博客结构。

本次改版采用用户确认的方案 A：尽可能参考 Cecilia Baldoni 个人站的信息架构、文字层级和内容展示方式，同时使用郭梦蕊自己的姓名、照片、中文内容和现有 Astro 代码实现。不复制参考站代码、插画或具体素材，只借鉴其排版气质和页面组织。

## 设计目标

- 全站品牌改为“郭梦蕊的博客”。
- 首页主标题使用“郭梦蕊”，不要再写“糖果小镇”作为站点主题。
- 首页上半部分参考 Cecilia 首页：姓名大标题、个人照片、几段个人介绍、一个引导用户探索内容的强调句。
- 首页下半部分保留互动地图作为导航入口，但文案改为“探索我的内容”一类的博客入口，不再强调糖果小镇世界观。
- About 页面使用用户提供的照片作为正式个人照，内容可先用自然、可信的占位介绍。
- Projects 页面参考 Cecilia 的项目 gallery：图片/图标 + 标题 + 简介的网格卡片。
- Blog 页面参考 Cecilia 的文章列表：标题、标签、摘要、日期，并在右侧或上方提供缩略图区域。
- 继续保留 Life 和 Contact 页面，但整体视觉语言要和 Blog / Projects 统一。

## 非目标

- 不一比一复制 Cecilia 网站的 HTML、CSS、插画、图标、照片或文字。
- 不继续强化“糖果小镇”作为站点名称或页面叙事。
- 不在本次重做新的 AI 城市地图；先复用现有地图资产作为临时互动地图。
- 不在第一版实现搜索、分类筛选、动态加载或评论系统。
- 不要求 About 内容完全真实完整；第一版先做格式和占位文字，后续再替换。

## 内容架构

主导航保持四到五个核心入口，命名更接近参考站：

- About：关于我。
- Projects：项目。
- Blog：博客。
- Life：生活。
- Contact：联系。

中文页面可显示中文标题，但导航和眉题可以保留英文，形成参考站类似的轻量学术/个人博客气质。

站点名称和页面标题规则：

- 站点名：郭梦蕊的博客。
- 首页标题：郭梦蕊。
- 页面标题示例：Blog | 郭梦蕊的博客。

## 首页设计

首页分为两个主要区块。

### 个人介绍区

结构参考 Cecilia 首页：

- `h1` 显示“郭梦蕊”。
- 左侧或正文中浮动显示用户提供的照片。
- 右侧或环绕文字区域放 2 到 3 段个人介绍。
- 文案暂时以占位方式表达：关注技术、AI 工具、项目实践、写作、生活记录和长期积累。
- 最后放一句加粗引导语，例如：“向下探索我的博客、项目、生活记录和联系方式。”

照片使用方式：

- 源照片来自 `/Users/xiaoru/Pictures/Photos Library.photoslibrary/originals/4/487FB75E-10D8-48A2-BD67-A5B98A3F4246.jpeg`。
- 复制到项目公开资源目录，生成网页用压缩版本。
- 首页使用接近参考站的矩形个人照，不做过度卡通化。
- About 页可以使用同一张照片的更正式头像/个人照样式。

### 探索区

保留现有互动地图资产，但修改所有可见文字：

- 区块标题：Explore / 探索我的内容。
- 地图热点标签：About、Projects、Blog、Life、Contact。
- 辅助文字入口也使用同一套命名。
- 地图 alt 文案不再写“糖果小镇”，改成“手绘互动地图导航”。

## About 页面

About 页要像一个正式个人主页，而不是模板占位卡片。

结构：

- 页面图标或小标题：About。
- 标题：关于我。
- 顶部使用照片 + 个人简介的两栏布局。
- 下方使用几个清晰小节：现在关注、正在建设、关键词。

占位内容方向：

- 郭梦蕊正在搭建一个长期记录技术、项目和生活的网站。
- 关注 AI 工具、工程效率、内容表达、个人知识管理和实践复盘。
- 网站会逐步沉淀技术文章、项目实验、生活记录和联系方式。

## Projects 页面

Projects 页面参考 Cecilia 的 `research-gallery` / `project-card` 形式。

每个项目卡片包含：

- 缩略图或图标。
- 项目标题。
- 1 到 2 句简介。
- 状态或标签。

第一版只有一个真实项目“郭梦蕊的博客”，因此需要补充 3 到 5 个占位项目卡片，避免页面显得空：

- 郭梦蕊的博客。
- AI 工具实践。
- Markdown 内容系统。
- 项目复盘笔记。
- 个人知识库。

占位项目可以先不建立详情页，或链接到现有项目详情页/未来占位。

## Blog 页面

Blog 页面参考 Cecilia 的文章 listing，而不是普通整块卡片。

每篇文章行包含：

- 标题。
- 标签。
- 摘要。
- 日期。
- 右侧缩略图或占位图。

目前只有一篇示例文章，可以继续显示，但页面结构要支持后续多篇 Markdown 自动扩展。标签显示要轻量，避免过多粉色装饰。

## 视觉语言

整体从“梦幻糖果主题”调整为“温暖、手绘、个人博客、轻学术感”：

- 背景：奶油白或淡米色。
- 主色：深绿色或墨绿色，参考 Cecilia 的沉稳导航色。
- 点缀：暖棕、金色、少量粉色。
- 字体：标题更接近 monospace / editorial feel；中文使用系统字体保持清晰。
- 卡片：不要过度玻璃拟态；用白底、细边框、轻阴影。
- 圆角：控制在 6 到 10px，避免糖果胶囊感过强。

## 实现范围

需要修改：

- `src/layouts/BaseLayout.astro`：站点名、导航文字、默认 description。
- `src/pages/index.astro`：首页个人介绍结构、照片、地图文案。
- `src/pages/about.astro`：照片和正式 About 结构。
- `src/pages/projects/index.astro`：项目 gallery。
- `src/pages/posts/index.astro`：Blog listing。
- `src/styles/global.css`：整体视觉语言、首页、About、Projects、Blog 样式。
- `src/content/projects/personal-website.md`：项目标题和摘要从糖果小镇改成个人博客。
- 必要时新增 `public/assets/profile/` 下的照片压缩文件。

## 验收标准

- 全站不再出现“糖果小镇”作为站点名称或主要文案。
- 首页第一屏能看到“郭梦蕊”、用户照片和个人介绍。
- 站点标题显示“郭梦蕊的博客”。
- 首页仍有互动地图入口，五个热点能进入正确页面。
- About 页有照片和个人介绍结构。
- Projects 页呈现图片/图标 gallery，而不是普通列表。
- Blog 页呈现接近参考站的文章 listing，而不是大块糖果风卡片。
- `npm run build` 通过。
- 本地浏览器检查首页、About、Projects、Blog、Contact 无明显布局溢出。

# 读书手帐 Studio 视觉设计规格

## 背景

现有 `/life/studio` 已经有一版双页 B5 读书手帐编辑器，但当前方向需要重新对齐：先完成高保真视觉设计，再进入实现。新版本不再做双页书本，也不预设固定的书名、摘抄、感想版块，而是一张可自由拼贴的竖向 B5 手帐页。

本规格只定义视觉、交互、字体、素材来源和发布边界。它不包含实现计划，也不要求此步骤修改现有 `/life/studio` 代码。

## 已确认决策

- 设计路径选择“设计稿优先”，先把首页和编辑器视觉定准，再实现。
- 页面是单张竖向 B5，不是 A5，也不是双页书本。
- 页面内容是自由空白画布，不预设固定信息结构。
- 审美主轴是日系干净文学感。
- 文本工具按 Canva 式可视化文本框设计。
- 左侧素材库按轮播工具箱设计，支持点击加入和拖拽加入。
- 右侧相册是“我的相册”，保存用户本地上传过的图片供重复使用。
- `保存草稿` 保存到本地或网页草稿。
- `发布` 直接生成网站公开可看的读书手帐页。

## 设计目标

- 首页像真实书桌上选纸张，而不是普通表单或素材列表。
- 编辑器第一眼能看懂：左边选素材，中间做手帐，右边调图片和文字。
- 手帐页保持日系、安静、文学、纸感，不走儿童贴纸或糖果 Y2K 风。
- 素材库必须可持续添加，不靠硬编码一堆按钮。
- 内置字体和素材必须有来源、授权、分类和默认尺寸记录。
- 公开发布页使用成品布局，不把第三方素材包装成可下载素材库。

## 非目标

- 第一版不做真正读取 iPhone 相册的系统级集成，只做本地上传和本地相册复用。
- 第一版不做多人协作、登录、云同步或在线素材后台。
- 第一版不从 Pinterest、小红书抓图作为内置公开素材。
- 第一版不做复杂 Markdown 编辑器；Markdown 只作为发布内容的附加正文或元数据。
- 第一版不做多页本子管理，只做一张 B5 页面。
- 第一版不把设计重心放在素材数量，而是放在视觉统一和可扩展素材结构。

## 整体视觉方向

关键词：

- 日系干净文学感
- 浅色书桌
- 米白纸张
- 低饱和绿、灰、米、淡棕
- 书、咖啡、花枝、邮票、纸夹
- 真实纸张、硫酸纸、牛皮纸、淡网格
- 安静、可读、留白多

避免：

- 高饱和粉紫蓝渐变
- 过多卡通图标
- 大面积糖果色
- 儿童贴纸感
- 过满的拼贴背景
- 素材按钮像后台管理系统

## 首页设计

首页是“选纸张”的书桌场景。

布局：

- 整屏浅木纹或暖灰桌面。
- 中央放一张竖向 B5 纸张预览。
- 纸张周围有少量真实书桌装饰：书脊、钢笔、纸夹、便签边角、咖啡杯阴影。
- 左右箭头用于翻看不同纸张。
- 底部按钮为 `选择这张纸`。
- 顶部可以保留网站导航，但视觉上不抢纸张预览。

首批底纸：

| id | 名称 | 质感 | 用途 |
| --- | --- | --- | --- |
| `paper-warm-cream` | 米白纸纹 | 暖白、轻纸纹 | 默认自由页 |
| `paper-soft-grid` | 浅灰网格 | 细网格、低对比 | 读书摘抄 |
| `paper-dot-note` | 淡点阵 | 极淡点阵 | 自由排版 |
| `paper-fine-line` | 细横线 | 淡灰横线 | 长段文字 |
| `paper-kraft-light` | 轻牛皮纸 | 淡棕纸纹 | 复古一点的读书页 |
| `paper-vellum` | 硫酸纸感 | 半透明白灰 | 叠放照片和摘抄 |

首页成功标准：

- 用户一眼知道是在选 B5 纸张。
- 纸张宽度不能显得像窄小 A5。
- 装饰服务于“在书桌上做手帐”，不能变成背景噪音。

## 编辑器设计

编辑器是单屏工作台。

```text
顶部：操作栏
左侧：轮播素材工具箱
中间：竖向 B5 画布
右侧：我的相册 + 文字设置 + 图层/属性
底部：保存草稿 + 发布
```

### 顶部操作栏

操作栏必须始终可见，放置高频编辑动作：

- 选择/移动
- 添加文字
- 上传图片
- 撤销
- 重做
- 上移一层
- 下移一层
- 复制
- 删除
- 缩放视图

按钮优先使用图标和短 tooltip，不使用大段说明文字。

### 中间 B5 画布

- 固定竖向 B5 比例。
- 默认显示用户在首页选择的底纸。
- 画布外有浅色阴影，像纸放在桌面上。
- 元素放入画布后支持移动、缩放、旋转、复制、删除和调整层级。
- 选中元素时显示控制点和旋转手柄。
- 画布内部不出现固定模板字段，所有内容由用户自由摆放。

### 左侧素材工具箱

左侧不是普通 grid，而是“工具箱 + 小轮播”。

每个素材区都有：

- 分类标题
- 左右箭头
- 当前素材大预览
- 小圆点或缩略位提示当前页
- `点击加入` 和拖拽加入两种操作

### 素材 Sheet 浏览验证

在进入真实编辑器实现前，独立视觉稿必须先验证素材工具箱对真实文件的表现：

- 构建时读取 `public/assets/life-journal/sheets/<sheet-id>/sheet.json`，不把具体素材名称硬编码进页面。
- 一个 sheet 一次显示四个缩略图，左右按钮翻该 sheet 内页；sheet 标签切换不同来源和风格包。
- 点击缩略图只更新视觉稿里的选中预览和 B5 页面示例，不在这一阶段写入草稿或改变现有 `/life/studio`。
- 只有素材文件、来源和许可说明完整的 sheet 才会出现在工具箱；候选来源不能作为可用素材显示。

首批分类：

- 便签本
- 胶带
- 贴纸
- 背景纸
- 文字样式
- 我的相册入口

### 右侧面板

右侧面板分三块：

1. 我的相册
   - 上传本地图片。
   - 展示最近上传图片。
   - 点击或拖拽图片到 B5 画布。
   - 图片保存在浏览器本地素材库，供下次继续使用。

2. 文字设置
   - 字体
   - 字号
   - 颜色
   - 对齐
   - 行距
   - 字距
   - 透明度
   - 旋转
   - 纹理风格，例如普通、钢笔、铅笔、淡墨。

3. 图层/属性
   - 当前选中元素名称。
   - X/Y 位置。
   - 宽高。
   - 旋转角度。
   - 层级顺序。

## 文字系统

文字系统由“文字预设卡片 + Canva 式右侧细调”组成。

首批文字预设：

| 预设 | 字体方向 | 用途 |
| --- | --- | --- |
| 正文摘抄 | 霞鹜文楷 | 长段读书摘抄 |
| 书名标题 | Noto Serif SC / 思源宋体风格 | 书名、作者、章节标题 |
| 手写金句 | Ma Shan Zheng | 一句话重点、页首句 |
| 标签小字 | Smiley Sans / 得意黑 | 日期、tab、短标签 |
| 批注便签 | 霞鹜文楷小字号 | 便签上的感想 |
| 英文引用 | 衬线 italic | 英文摘抄或页眉 |

字体来源要求：

| 字体 | 来源 | 设计用途 | 实现阶段要求 |
| --- | --- | --- | --- |
| LXGW WenKai / 霞鹜文楷 | https://github.com/lxgw/LxgwWenKai | 正文、批注 | 下载前核对仓库 LICENSE，优先自托管 woff2 |
| Smiley Sans / 得意黑 | https://github.com/atelier-anchor/smiley-sans | 标签、标题点缀 | 下载前核对 Releases 和 LICENSE，优先自托管 woff2 |
| Ma Shan Zheng | https://fonts.google.com/specimen/Ma+Shan+Zheng | 手写金句 | 使用 Google Fonts 或下载自托管，记录 license |
| Noto Serif SC | https://fonts.google.com/noto/specimen/Noto+Serif+SC | 书名、正文衬线 | 使用 Google Fonts 或下载自托管，记录 license |
| Noto Sans SC | https://fonts.google.com/noto/specimen/Noto+Sans+SC | UI 面板 | 使用 Google Fonts 或系统字体 fallback |

字体 UI：

- 左侧提供文字预设卡片，点击后在画布创建文本框。
- 右侧显示当前文本框的细调项。
- 字体列表只显示已安装或已加载的字体，避免用户选到不可渲染字体。
- 发布时必须保证公开页面能加载同样字体或有可靠 fallback。

## 素材来源与授权约束

Pinterest、小红书只能作为灵感来源，不能直接抓图作为网站内置素材。

内置素材优先来源：

| 来源 | 链接 | 可用方向 | 约束 |
| --- | --- | --- | --- |
| Illustcenter | https://illustcenter.com/en/ | 文具、书本、花、咖啡、标签等插图 | 条款允许个人/公司、商业/非商业免费使用；一次使用数量有限制；禁止素材再分发和把素材作为模板素材销售 |
| DigitalScrapbook | https://www.digitalscrapbook.com/ | 纸纹、背景纸、贴纸、拼贴 PNG | 必须逐个素材核对 Personal/Commercial 授权，不能公开分发原始素材文件 |
| Google Fonts | https://fonts.google.com/ | 字体 | 使用前记录具体字体 license 和加载方式 |
| 字体/素材 GitHub 官方仓库 | 各字体仓库 | 开源字体 | 以仓库 LICENSE 和 Releases 为准，不依赖第三方转载下载 |

实现时必须建立素材 manifest。每个素材至少记录：

```json
{
  "id": "tape-warm-paper-01",
  "category": "胶带",
  "stylePack": "japanese-literary",
  "name": "米色纸胶带",
  "src": "/assets/life-journal/japanese-literary/tapes/warm-paper-01.png",
  "preview": "/assets/life-journal/japanese-literary/tapes/warm-paper-01-thumb.webp",
  "sourceUrl": "https://illustcenter.com/en/",
  "license": "Illustcenter terms checked on import; free for individual/company commercial and non-commercial use within site limits; redistribution prohibited",
  "defaultWidth": 180,
  "defaultHeight": 34,
  "tags": ["paper", "warm", "quiet"]
}
```

素材规则：

- 没有来源链接和授权说明的素材不能内置。
- 第三方素材可以用于公开成品展示，但不能把原始素材文件暴露成“素材下载包”。
- 需要压缩大图，并为工具箱生成缩略图。
- 上传到“我的相册”的用户图片只保存在本地 IndexedDB，不进入公开素材库，除非用户发布当前作品时明确把它写入该作品资源目录。

## 保存与发布

### 保存草稿

- 保存当前 B5 画布 JSON。
- 保存用户上传图片 blob。
- 保存所选底纸、字体和页面元数据。
- 保存到浏览器本地 IndexedDB。
- 保存后不影响公开网站内容。

### 发布

发布是本地开发环境中的内容生成动作：

- 生成 `src/content/life/<slug>.md`。
- 生成 `public/assets/life/<slug>/layout.json`。
- 将本次作品用到的用户上传图片复制到 `public/assets/life/<slug>/`。
- 公开页面 `/life/<slug>/` 可以直接渲染这张手帐。
- 发布不会创建公开编辑入口。

发布前需要检查：

- 标题、日期、slug 是否有效。
- 作品中用到的字体是否能在公开页加载。
- 第三方素材是否来自 manifest，并有授权记录。
- 用户上传图片是否已复制到作品资源目录。

## 视觉设计稿交付范围

设计稿优先阶段只需要两屏：

1. 首页：书桌选纸张。
2. 编辑器：单张 B5 自由画布 + 左侧轮播素材工具箱 + 右侧相册/文字/属性面板。

设计稿需要显示：

- 首页纸张翻页状态。
- 至少一张 B5 页面空白状态。
- 至少一张带少量素材的编辑状态。
- 左侧便签、胶带、贴纸轮播的具体样子。
- 右侧文字设置面板的具体字段。
- `保存草稿` 和 `发布` 的位置。

## 验收标准

- 用户看到首页能判断这是“在书桌上选一张 B5 手帐纸”。
- 用户看到编辑器能判断可以自由拖拽素材做一张读书手帐。
- 整体气质是日系干净文学感，而不是儿童贴纸、糖果 Y2K 或后台工具。
- 字体和素材来源有明确采购/下载路径。
- 素材系统支持后续持续添加。
- 设计稿不会承诺第一版无法实现的 iPhone 相册直连、云同步或在线素材市场。
- 规格不依赖 Pinterest/小红书图片作为内置素材。

## 后续步骤

1. 根据本规格制作高保真视觉稿或静态 mockup。
2. 用户确认视觉稿后，再写实现计划。
3. 实现计划通过后，才修改 `/life/studio`。
4. 实现完成后必须用浏览器检查素材加入、文字编辑、层级、保存和发布流程。

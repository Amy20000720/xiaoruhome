# 中文个人网站设计方案

## 背景

目标是构建一个中文个人网站，用手绘风格表达“全方位记录和展示自己”的入口。参考方向来自 Cecilia Baldoni 的互动个人站，但本项目不复制城市地图，而采用更贴合个人空间感的“收藏柜工作室”首页。

网站第一版应轻量、稳定、容易长期维护。它应优先服务个人介绍、技术文章、项目入口、个人生活记录和联系方式，而不是做成复杂的动态应用。

## 设计目标

- 用一屏完整展示的手绘收藏柜作为首页主视觉。
- 让访问者能一眼理解五个主要入口：关于我、技术文章、项目入口、个人生活、联系我。
- 使用中文作为主要内容语言。
- 使用真实网页文本承载导航标签和正文，避免把中文文字烘焙进图片。
- 支持后续持续添加文章、项目和生活记录。
- 第一版保持实现简单，避免过重的交互和框架复杂度。

## 非目标

- 第一版不做多语言站点。
- 第一版不做复杂 3D、长滚动叙事或游戏化房间探索。
- 第一版不要求后台管理系统。
- 第一版不把所有内容塞进首页，首页只负责清晰导览。

## 推荐技术路线

采用 Astro 作为静态网站框架，使用 Markdown 或 MDX 管理文章、项目和生活记录。

选择 Astro 的原因：

- 适合静态个人网站，构建结果轻量。
- 比 Quarto 更适合自由视觉首页和自定义交互。
- 比 Next.js 更轻，第一版维护成本更低。
- Markdown 内容流适合长期写技术文章和生活记录。
- 可部署到 GitHub Pages、Vercel 或 Netlify。

## 信息架构

第一版包含五个一级入口：

- `/about`：关于我。展示个人介绍、当前关注方向、经历和技能关键词。
- `/posts`：技术文章。展示技术文章列表，文章使用 Markdown 或 MDX 编写。
- `/projects`：项目入口。展示项目卡片、项目说明、技术栈和外部链接。
- `/life`：个人生活。展示照片、随笔、书影音、旅行或其他生活记录。
- `/contact`：联系我。展示邮箱、GitHub、社交链接和合作说明。

首页 `/` 是收藏柜入口页，不承担长文本内容。

## 首页视觉设计

首页是一屏完整展示的手绘收藏柜工作室。收藏柜中的五个物件对应五个主要栏目：

- 相框或名牌：关于我。
- 书本或笔记本：技术文章。
- 工具箱、电脑或文件夹：项目入口。
- 照片盒或小摆件：个人生活。
- 信封、电话或留言卡：联系我。

视觉风格应满足：

- 手绘线条清晰，有纸张质感或轻微不规则边缘。
- 颜色温暖但不单调，避免只有单一色系。
- 首页物件之间要有足够空间，避免热点重叠。
- 中文标签由网页文本渲染，hover 或移动端固定显示。
- 图片不承担精确中文文字，减少图像模型文字错误。

## 交互设计

桌面端：

- 鼠标悬停物件时，物件轻微上浮或高亮。
- 悬停时显示中文标签和一句短说明。
- 点击物件进入对应页面。

移动端：

- 收藏柜仍保持一屏完整展示。
- 导航标签可以固定显示，保证可读和可点。
- 热点区域需要足够大，避免只能精确点击小图标。
- 如果竖屏空间不足，可以在收藏柜下方显示同样的五个文字入口，但首页主视觉仍保持完整。

## 手绘资产生成流程

手绘资产采用迭代式生成，不直接一次性生成最终全站：

1. 生成 3 到 5 张收藏柜风格探索图，确认线条、透视、色彩和质感。
2. 选定风格后生成首页收藏柜主图。
3. 生成或裁切五个入口物件的局部资产，用于 hover 高亮或后续页面装饰。
4. 对图片做压缩和格式转换，优先使用 WebP，必要时保留 PNG 源文件。
5. 前端用 CSS 定位热点和真实中文标签。

图片生成提示词应明确：

- 中文个人网站首页。
- 手绘收藏柜工作室。
- 五个可识别物件：相框、书本、工具箱或电脑、照片盒、信封。
- 无文字、无水印、无 logo。
- 清晰线稿、温暖配色、网页插画、正面或轻微透视。

## 页面内容设计

`/about` 页面：

- 简短自我介绍。
- 当前关注方向。
- 技能、兴趣和价值观关键词。
- 可选时间线。

`/posts` 页面：

- 文章列表。
- 分类或标签。
- 每篇文章包含标题、日期、摘要和正文。

`/projects` 页面：

- 项目卡片列表。
- 每个项目包含名称、说明、技术栈、状态和链接。
- 后续可扩展为项目详情页。

`/life` 页面：

- 生活记录列表。
- 支持图文混排。
- 内容可以比技术文章更轻松，但仍保持统一视觉。

`/contact` 页面：

- 邮箱。
- GitHub 或其他技术主页。
- 社交链接。
- 简短合作说明。

## 测试和验收

第一版验收标准：

- 首页在桌面端一屏完整展示，不需要滚动才能看到五个入口。
- 五个入口都可点击并进入正确页面。
- 中文标签清晰可读，不依赖图片内文字。
- 移动端入口可读、可点，不互相遮挡。
- 技术文章、项目和生活记录可以通过 Markdown 新增。
- 构建命令可以成功执行。
- 首页图片资源经过压缩，首屏加载不过重。

## 后续扩展

- 增加文章搜索。
- 增加项目筛选。
- 增加深色模式。
- 增加更细的页面插画。
- 增加 RSS。
- 增加站点统计，但应优先考虑隐私和性能。

# 读书手帐 Studio：可编辑与本地发布设计

**日期：** 2026-07-13

**状态：** 待用户确认

**适用路由：** `/life/studio/`（本地编辑器）与 `/life/<slug>/`（公开成品）

## 1. 目标

把现有的演示型 Studio 重构成可完成一张真实读书手帐页的本地工具。用户在本机打开 `/life/studio/`，从按 Sheet 分类的真实素材库中挑选贴纸、胶带和插图，上传自己的照片、添加并编辑文字，再把所有元素自由重叠、移动、缩放、旋转、调整层级。完成后可保存为浏览器本地草稿，或直接发布为这个个人网站中的一篇公开 Life 页面。

成品是**一张竖版 B5 读书手帐页**，不是双页本子，也不是模板选择器。视觉方向是日系、干净、安静、有文学感，页面保留足够空白让用户自己完成拼贴。

## 2. 已确认边界

- 编辑器只在本地开发环境提供；部署到 GitHub Pages 后不暴露可编辑素材库或写文件能力。
- 编辑、草稿与发布均以本机为中心，不引入账号、云同步或数据库服务。
- 素材以真实下载文件为准，按 Sheet 浏览；不再用 CSS 方块假装贴纸或胶带。
- Studio 中可使用的插图必须保留来源和许可记录。Illustcenter 素材只供 Studio 页面内使用，不提供原文件下载；CC0 素材可进入公开成品。
- 发布操作只写入指定的 Life Markdown、布局 JSON 和本次上传的图片，不改动任何其他文章或内容。

## 3. 使用流程

1. 本地打开 Studio，默认进入一张空白但带轻量纸张纹理的竖版 B5 画布。
2. 从左侧素材工具箱选择 Sheet，再以前后箭头翻页挑选素材；点击即可加入画布。
3. 使用顶部操作栏移动、缩放、旋转、复制、删除，以及前移、后移、置顶、置底。
4. 用文字工具新建文字，选择字体、字号、颜色、对齐方式和基础字重；双击画布上的文字即可改写。
5. 从本机上传照片或图片，将其插入画布并采用同一套变换能力。
6. 填写标题、日期、摘要、标签等发布信息。
7. 点击“保存草稿”将画布、元数据和上传图片存进 IndexedDB。
8. 点击“发布”后，若相同 slug 已存在，先明确提示将覆盖该文章的 Studio 生成内容；确认后本地开发服务器写入公开页面文件，并显示成品链接。

## 4. 页面与交互结构

### 4.1 本地 Studio

桌面端采用三栏工作台，信息密度优先，不把每一栏再嵌套成卡片：

- **左栏，素材工具箱**：Sheet 标签、上一页/下一页、真实缩略图网格、上传图片入口、文字工具入口。
- **中栏，画布工作区**：逻辑尺寸固定为 `420 x 594`，与 B5 纵向比例一致；显示尺寸可随视口缩放，但对象坐标永远保存为逻辑尺寸。顶部为操作栏，底部为“保存草稿”和“发布”。
- **右栏，属性与发布信息**：选中元素时显示位置、宽高、旋转、透明度和层级命令；未选中时以文章元数据和文字默认样式为主。

移动端按“工具箱、画布、属性/发布”纵向排列。画布按容器缩放，页面不得因 B5 画布产生横向滚动。

### 4.2 操作栏

操作栏保留明确、可见的命令，而不是依赖隐蔽快捷键：

- 指针选择、添加文字、上传图片
- 复制、删除
- 前移、后移、置顶、置底
- 缩放数值、旋转数值、透明度
- 撤销、重做只在 Fabric 当前版本能稳定记录对象变更时加入；否则本期不承诺伪撤销功能。

画布对象统一使用 Fabric 的控制点：角点缩放、旋转手柄旋转、直接拖拽移动。照片、素材和文字的层叠关系必须在发布布局中保留。

### 4.3 素材 Sheet

Studio 读取 `public/assets/life-journal/sheets/*/sheet.json` 作为唯一素材目录。每个 manifest 至少包含名称、来源、许可说明和素材数组。素材浏览遵循：

- 一个 Sheet 一次显示固定数量缩略图，左右箭头只切换当前 Sheet 的页码。
- 切换 Sheet 时重置页码并更新可插入素材。
- 每个缩略图保留来源名和许可提示，避免以后无法追溯。
- 用户新增的素材不混进下载素材 Sheet；上传图片属于当前草稿，仅在发布时写入当前文章目录。

第一批使用现有两个已核验 Sheet：`sweets-line-art-color` 与 `botanical-heritage-cc0`。后续只需将新的、许可清楚的 Sheet 放入相同目录和 manifest，即可自动进入工具箱，无需重写编辑器。

## 5. 字体与文字

文字不是预先画好的贴纸。Studio 使用可编辑文本对象，最初提供清晰衬线、干净无衬线和两种手写感字体，且仅在字体文件实际加载成功时出现在选择器中。推荐首批组合：

- 正文：Noto Serif SC 或等效中文衬线字体。
- 标签与信息栏：Noto Sans SC 或等效中文无衬线字体。
- 标题与短句：LXGW WenKai。
- 强调小字：Ma Shan Zheng 或另一个许可清楚、已自托管的中文手写字体。

用户可选择颜色、字号、行高、对齐和基础字重。导出时保存字体 family 与完整文字样式；公开页面加载同一字体或可读的回退字体，不能把文字烘焙成图片。

## 6. 草稿与发布数据

### 6.1 草稿

IndexedDB 延续现有 `xiaoru-life-journal` 数据库，但升级保存结构以记录：

- `page.size: "b5"` 与 `orientation: "portrait"`
- Fabric canvas JSON、文章元数据和当前选中的样式
- 上传照片的数据 URL，确保本机刷新后仍可恢复
- 最近一次保存时间与草稿版本

旧双页草稿不自动迁移为新 B5 页面，避免坐标和比例错误；用户可保留旧数据，但 Studio 会将其标为旧格式并提供新建 B5 页入口。

### 6.2 发布

静态 Astro 站点无法从部署后的浏览器写入仓库。因此发布功能只在 `astro dev` 时由 Vite 开发服务器中间件提供，端点采用站点 base-aware 路径，例如：

`POST /xiaoruhome/__journal_publish`

客户端发送元数据、B5 布局与本次上传图片。中间件验证输入后只可写入：

- `src/content/life/<safe-slug>.md`
- `public/assets/life/<safe-slug>/layout.json`
- `public/assets/life/<safe-slug>/upload-<id>.<extension>`

slug 仅允许小写字母、数字和连字符，拒绝路径分隔符、`..`、未知 MIME、过大的 JSON 或图片。上传图片由中间件解码为真实文件，并将画布布局中的 data URL 替换为站点可访问路径。

同一 slug 再次发布时，仅覆盖该 slug 的 Markdown、布局和以 `upload-` 前缀命名的 Studio 托管上传图；不删除其他手动放入的资源文件。发布成功后中间件返回实际公开路径，开发服务器重新读取内容，用户可立即在 `/life/<slug>/` 检查成品。

生产构建不注册此中间件，`/life/studio/` 只显示“本地编辑器仅在开发环境可用”的简短提示，不包含编辑器数据、上传控件或发布按钮。

## 7. 公开页面渲染

发布后的 Markdown frontmatter 使用现有 Life collection 字段，并标记 `draft: false`。布局 JSON 使用：

```json
{
  "page": {
    "size": "b5",
    "orientation": "portrait",
    "theme": "reading-journal-b5"
  }
}
```

公共渲染器沿用现有 `JournalPage` 和 `JournalElement`，增加 B5 纵向主题。元素的 `x/y/width/height` 继续按 `420 x 594` 换算为百分比，所以 Studio 中看到的重叠、比例、旋转和层级能直接复现于公开页面。新主题为暖白纸面、浅色方格或纸张纹理、低对比边框与克制的排版，不沿用旧粉色 Y2K 双页外观。

公开页面不会渲染 Studio 的素材分类、编辑命令、草稿信息或上传入口。

## 8. 实现范围

计划修改或新增的文件：

- `astro.config.mjs`：注册只在开发模式启用的本地发布中间件。
- `scripts/local-journal-publish.ts`：验证、文件写入和返回公开路径。
- `src/pages/life/studio.astro`：替换为单页 B5 Studio，并在开发/生产之间分支渲染。
- `src/scripts/journal-studio.ts`：改写为 B5 Fabric 画布、真实 Sheet 选择、文字/图片、层级和发布流程。
- `src/scripts/journal-storage.ts`：升级本地草稿版本与新页面格式。
- `src/lib/journalLayout.ts`、`src/lib/journalAssets.ts`：增加 B5 页面类型及共享素材 manifest 读取。
- `src/components/JournalPage.astro`、`src/components/JournalElement.astro`、`src/styles/global.css`：增加公开 B5 渲染主题。
- `src/styles/journal-studio.css`：Studio 工作台及响应式规则。

现有 `/life/reading-journal-studio-design/` 保留为视觉和素材浏览参考，不作为最终编辑器实现的一部分。

## 9. 验收标准

- 本地 `/life/studio/` 是一张清晰的竖版 B5 画布，不再出现双页横向画布。
- 可用 Sheet 标签和左右箭头浏览真实缩略图；点击任一素材能加入画布。
- 可上传图片、插入编辑文字，并对所有对象拖动、缩放、旋转、改变层级、复制和删除。
- 刷新浏览器后可从 IndexedDB 恢复保存的 B5 草稿和上传图片。
- 发布后生成正确的 Markdown、`layout.json` 和上传图，公开 `/life/<slug>/` 与 Studio 画布构图一致。
- 发布端点拒绝路径穿越、非法 slug、未知图片类型和超限请求。
- `npm run build` 通过；生产构建中没有可用的本地文件写入端点。
- 使用桌面和移动视口截图检查：文字不溢出、画布不空白、工具箱翻页正常、页面无横向溢出。

## 10. 非目标

- 本期不做账号、多人协作、云同步、在线素材商店或 iPhone 相册直接接入。
- 本期不把 Pinterest、小红书等来源不明确的素材批量抓取进网站。
- 本期不提供原始第三方贴纸的公开下载或打包导出。
- 本期不实现完整 Canva 的所有图形编辑、模板市场或复杂排版系统；重点是可靠完成一张读书手帐页。

# A5 电子手帐生活页设计方案

## 背景

当前 `/life` 页面是普通生活记录列表，内容来自 `src/content/life/` Markdown 集合。它能长期维护，但视觉上还不像一个有个人气质的生活空间。

新的方向是把生活区升级为“公开展示的电子手帐书”。访问者看到的是精致的 A5 单页手帐作品；本地则提供一个轻量 Studio 编辑器，方便拖拽素材、照片和文字，导出成可提交到网站内容目录的记录。

这个方案保留现有 Astro 静态网站和 Markdown 内容流，不引入账号系统、数据库或在线公开编辑。

## 设计目标

- `/life` 从普通列表升级为电子手帐目录页。
- `/life/[id]` 展示一张完整 A5 单页手帐。
- 新增本地使用的 `/life/studio` 编辑器，用于拖拽素材、写日记、摆放照片和导出内容。
- 手帐素材丰富，但实现保持轻量、可维护。
- 保留 `title`、`date`、`summary`、`tags`、`mood`、`location` 等结构化字段。
- 文字内容使用真实 HTML/Markdown 渲染，不烘焙进图片。
- 视觉语言延续网站的手绘、糖果色、卡通、Y2K 个人风格。

## 非目标

- 第一版不做登录系统。
- 第一版不做云端保存或在线多人编辑。
- 第一版不做复杂无限白板。
- 第一版不把手帐页导出成唯一图片作为正式内容。
- 第一版不做完整素材后台管理。
- 第一版不改变现有文章、项目和生活记录的基本内容系统。

## 推荐版本

选择“公开展示 + 本地轻量编辑器”的版本。

公开站点负责展示成品：

- 访问者打开 `/life` 和 `/life/[id]`，看到的是设计完成的手帐页。
- 公开页面不显示编辑器入口，或仅在本地开发环境中显示编辑入口。

本地 Studio 负责创作：

- 用户在本机打开 `/life/studio`。
- 使用拖拽素材、照片上传、文字块和模板完成一张 A5 页面。
- 草稿保存在浏览器本地。
- 完成后导出 Markdown 和 layout JSON。
- 导出的文件再放回 `src/content/life/`，图片放到 `public/assets/life/`。

## 信息架构

### `/life`

生活区首页是电子手帐书目录，而不是普通卡片列表。

主要区域：

- 一本打开的手帐或手帐封面视觉。
- 最近一篇生活记录的精选手帐预览。
- 月份标签，例如 `2026.07`、`2026.08`。
- 心情、地点、主题标签索引。
- 若干生活记录缩略页，每个缩略页像贴在桌面上的小纸张。

### `/life/[id]`

单篇生活记录是一张完整 A5 手帐页。

页面结构：

- A5 纸张容器。
- 日期区。
- 标题区。
- 正文区。
- 照片区。
- 手帐装饰层，包括贴纸、胶带、便签、边框、日期章。
- 页面外侧保留返回目录、上一篇、下一篇入口。

### `/life/studio`

本地编辑器是单页 A5 手帐创作工具。

布局：

- 左侧素材抽屉。
- 中间 A5 画布。
- 右侧属性面板。
- 顶部工具栏。

主要操作：

- 添加文字块。
- 上传照片。
- 插入贴纸。
- 插入胶带。
- 插入便签纸。
- 调整元素位置、大小、旋转角度和层级。
- 保存本地草稿。
- 导出 Markdown 和 layout JSON。

## 内容模型

现有 life 集合继续使用 Markdown frontmatter。第一版扩展字段如下：

```yaml
---
title: "周末小记"
date: 2026-07-08
summary: "一张关于周末、咖啡和散步的手帐。"
tags:
  - 生活
  - 手帐
mood: "轻松"
location: "上海"
journalLayout: "/assets/life/2026-07-08-weekend/layout.json"
cover: "/assets/life/2026-07-08-weekend/cover.webp"
---
```

正文 Markdown 仍然承载可阅读、可搜索的日记文本。`journalLayout` 只负责记录视觉摆放信息。

第一版将 layout JSON 放在 `public/assets/life/<slug>/layout.json`。Astro 构建时可以读取该 JSON 生成静态展示页，浏览器也可以直接访问同一份资源。

layout JSON 示例：

```json
{
  "version": 1,
  "page": {
    "size": "a5",
    "orientation": "portrait",
    "theme": "candy-y2k-grid"
  },
  "elements": [
    {
      "id": "title-note",
      "type": "text",
      "text": "周末小记",
      "x": 68,
      "y": 82,
      "width": 260,
      "height": 70,
      "rotation": -3,
      "style": "pink-sticky-title"
    },
    {
      "id": "photo-1",
      "type": "image",
      "src": "/assets/life/2026-07-08-weekend/photo-1.webp",
      "x": 92,
      "y": 180,
      "width": 260,
      "height": 190,
      "rotation": 2,
      "style": "polaroid"
    }
  ]
}
```

## 素材系统

素材应优先使用 SVG、CSS 和少量压缩图片，避免大量不可控大图。

第一版内置素材分类：

- 纸张模板：网格纸、横线纸、点阵纸、奶油纸、糖果渐变纸。
- 胶带：格纹、波点、星星、半透明彩带、手绘边框。
- 贴纸：心情、天气、咖啡、书、电脑、花、旅行、日常小物。
- 便签：方形便签、撕边纸、拍立得卡片、日期卡。
- 装饰：月份标签、日期章、分隔线、小箭头、闪光符号。
- 照片框：拍立得、圆角相框、胶片框、撕纸框。

素材命名应稳定，例如：

```text
public/assets/journal/stickers/mood-happy.svg
public/assets/journal/tapes/pink-grid.svg
public/assets/journal/papers/cream-grid.svg
public/assets/journal/frames/polaroid.svg
```

## 视觉方向

关键词：

- A5 电子手帐。
- 手绘贴纸。
- 糖果色。
- 卡通但不幼稚。
- Y2K 小装饰。
- 生活感、私人收藏感、可爱但清晰。

颜色建议：

- 奶油白作为纸张底色。
- 糖果粉、薄荷蓝、香草黄、淡紫作为主装饰色。
- 少量亮蓝、荧光粉、银色或星光黄作为 Y2K 点缀。
- 正文区域必须保持高对比度，不能被装饰影响可读性。

页面质感：

- 纸张可以有轻微纹理、虚线、格线或撕边。
- 元素允许轻微旋转，制造手工拼贴感。
- 贴纸边缘可以有白色描边。
- 胶带可以半透明覆盖照片边角。
- 不使用把整个页面压成一张图的方式展示正文。

## 技术路线

继续使用 Astro 作为站点主体。

公开展示：

- `/life` 和 `/life/[id]` 使用 Astro 静态渲染。
- 从 Markdown frontmatter 和 layout JSON 读取内容。
- 通过 HTML/CSS 渲染 A5 手帐纸和元素层。
- 图片资源从 `public/assets/life/` 加载。

本地编辑器：

- `/life/studio` 使用一个原生客户端脚本，不要求引入 React 或 Preact。
- 第一选择是 Fabric.js，因为它适合固定页面上的拖拽、缩放、旋转和序列化。
- 不使用 tldraw 作为第一版核心画布，因为本需求不是无限白板。
- 不使用纯 dnd-kit 作为画布核心，因为它对自由旋转和缩放支持不足。

草稿保存：

- 使用 IndexedDB 保存草稿、layout 数据和上传图片 blob。
- `localStorage` 只保存最近打开的草稿 id 和轻量偏好。
- 导出时生成 Markdown 文本和 layout JSON 文本。

## 编辑器功能

第一版必须有：

- 固定 A5 竖版画布。
- 添加文字块。
- 上传并摆放图片。
- 插入内置贴纸。
- 插入内置胶带。
- 拖拽移动元素。
- 缩放元素。
- 旋转元素。
- 删除元素。
- 调整层级。
- 保存本地草稿。
- 导出 Markdown 和 layout JSON。

第一版可以没有：

- 自由画笔。
- 橡皮擦。
- 多页管理。
- 账号登录。
- 云端同步。
- 协作。
- 在线发布按钮。

## 响应式设计

公开页面：

- 桌面端 A5 页面居中展示，周围可以有桌面、贴纸、索引标签等装饰。
- 移动端 A5 页面按宽度缩放，正文仍然可读。
- 手帐元素不能遮挡导航和正文关键内容。

编辑器：

- 第一版优先支持桌面端编辑。
- 移动端可以显示只读提示或简化编辑模式。
- 素材面板和属性面板在窄屏下可以折叠。

## 验收标准

视觉验收：

- `/life` 一眼看起来像电子手帐目录，而不是普通文章列表。
- `/life/[id]` 一眼看起来像完整 A5 手帐页。
- 风格明显比普通博客页更手绘、彩色、贴纸化。
- 正文、日期、标题清晰可读。
- 贴纸和胶带丰富但不遮挡内容。

功能验收：

- 现有 life Markdown 记录仍能正常显示。
- 新增 layout JSON 后，公开页面能渲染手帐布局。
- `/life/studio` 能添加、拖拽、缩放、旋转、删除元素。
- 本地草稿刷新后仍能恢复。
- 导出内容可用于创建新的 `src/content/life/*.md` 和 layout JSON。
- `npm run build` 通过。

## 实施顺序

1. 扩展 life 内容模型，允许 `journalLayout` 字段。
2. 建立 `public/assets/journal/` 素材目录和第一批 SVG/CSS 素材。
3. 建立 layout JSON 读取和 A5 展示组件。
4. 改造 `/life` 为手帐目录页。
5. 改造 `/life/[id]` 为 A5 手帐展示页。
6. 新增 `/life/studio` 本地编辑器。
7. 实现导出 Markdown 和 layout JSON。
8. 用一篇示例生活记录完成端到端验证。
9. 检查桌面和移动端视觉效果。
10. 运行构建验证。

## 风险和处理

- 风险：编辑器复杂度过高。
  - 处理：第一版只做单页 A5，不做多页、不做云端、不做自由白板。
- 风险：素材越加越乱。
  - 处理：素材按类别和稳定命名组织，展示层只读取声明式数据。
- 风险：视觉元素遮挡正文。
  - 处理：正文块作为主要内容层，装饰素材默认避开正文区域。
- 风险：GitHub Pages 不能保存编辑结果。
  - 处理：明确采用本地草稿和导出文件，不承诺在线保存。
- 风险：移动端编辑体验差。
  - 处理：第一版优先桌面编辑，移动端保证公开展示体验。
